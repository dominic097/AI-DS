# Gradient descent Algorithm 
Gradient descent is an optimization algorithm used to minimize a function by iteratively moving towards the steepest descent / minimize loss, which is the negative of the gradient. 

It is commonly used in machine learning and deep learning to optimize loss functions.

Gradient descent finds the best weights (parameters) for a model by minimizing the loss function, which measures how well the model's predictions match the actual data.

Weight and bais es are updated using the following formulas:

```
weight = weight - learning_rate * dL/dweight
bias = bias - learning_rate * dL/dbias
```

where:
- `weight` and `bias` are the parameters of the model.
- `learning_rate` is a hyperparameter that controls the step size during each iteration.
- `dL/dweight` and `dL/dbias` are the gradients of the loss function with respect to the weight and bias, respectively.

### Batch Gradient Descent:
In batch gradient descent, the entire dataset is used to compute the gradient of the loss function for each iteration. This approach can be computationally expensive for large datasets but provides a stable convergence.

Algorithm Steps:
1. Initialize weights and biases randomly.
2. For each iteration:
    - Compute the predictions for the entire dataset.
    - Calculate the loss using the loss function.
    - Compute the gradients of the loss with respect to weights and biases.
    - Update weights and biases using the gradient descent update rules.
3. Repeat until convergence or for a fixed number of iterations.

### Example Code for Batch Gradient Descent

```python
import  numpy as np
X = np.array([1,2,3,4])
Y = np.array([2,4,6,8])

w=0
lr=0.01

# Training loop: iterate for 1000 epochs
for epoch in range(1000):
    Y_pred = X * w
    
    # Calculate loss and gradients
    loss = np.mean((Y - Y_pred) ** 2)
    
    #Compute gradients
    dw = (-2 / len(X)) * np.sum(X * (Y - Y_pred))

    w = w - lr * dw
    if epoch % 100 == 0:
        print(f'Epoch {epoch}, Loss: {loss}, Weight: {w}')  

```

# Stochastic Gradient Descent (SGD):
In stochastic gradient descent, a single data point (or a small batch of data points) is used to compute the gradient of the loss function for each iteration. This approach is computationally efficient and can lead to faster convergence, but it may introduce more noise in the updates.

Algorithm Steps:
1. Initialize weights and biases randomly.
2. For each iteration:
    - Randomly select a single data point (or a small batch).
    - Compute the prediction for the selected data point.
    - Calculate the loss using the loss function.
    - Compute the gradients of the loss with respect to weights and biases.
    - Update weights and biases using the gradient descent update rules.
3. Repeat until convergence or for a fixed number of iterations.    

### Example Code for Stochastic Gradient Descent
```python
import numpy as np
X = np.array([1,2,3,4])
Y = np.array([2,4,6,8])

w=0 
lr=0.01

# Training loop: iterate for 1000 epochs
for epoch in range(1000):
    for i in range(len(X)):
        Y_pred = X[i] * w
        loss = (Y[i] - Y_pred) ** 2
        dw = -2 * X[i] * (Y[i] - Y_pred)
        w = w - lr * dw
    if epoch % 100 == 0:
        print(f'Epoch {epoch}, Loss: {loss}, Weight: {w}')  
```

4) Reinforcement Learning: In reinforcement learning, SGD can be used to optimize policy and value functions based on individual experiences or episodes.

# Mini-batch Gradient Descent:
In mini-batch gradient descent, a small batch of data points is used to compute the gradient of the loss function for each iteration. This approach balances the computational efficiency of stochastic gradient descent with the stability of batch gradient descent.

Algorithm Steps:
1. Initialize weights and biases randomly.
2. For each iteration:
    - Randomly select a small batch of data points.
    - Compute the predictions for the selected batch.
    - Calculate the loss using the loss function.
    - Compute the gradients of the loss with respect to weights and biases.
    - Update weights and biases using the gradient descent update rules.
3. Repeat until convergence or for a fixed number of iterations.

### Example Code for Mini-batch Gradient Descent

```python
import numpy as np
X = np.array([1,2,3,4]) 
Y = np.array([2,4,6,8])
w=0
lr=0.01
batch_size = 2  
# Training loop: iterate for 1000 epochs
for epoch in range(1000):
    for i in range(0, len(X), batch_size):
        X_batch = X[i:i+batch_size]
        Y_batch = Y[i:i+batch_size]
        Y_pred = X_batch * w
        loss = np.mean((Y_batch - Y_pred) ** 2)
        dw = (-2 / len(X_batch)) * np.sum(X_batch * (Y_batch - Y_pred))
        w = w - lr * dw
    if epoch % 100 == 0:
        print(f'Epoch {epoch}, Loss: {loss}, Weight: {w}')
```

Example Scenarios
1) Image Classification: Mini-batch gradient descent is commonly used in training deep learning models for image classification tasks, where large datasets like ImageNet are processed in smaller batches.
2) Speech Recognition: In speech recognition tasks, mini-batch gradient descent can be used to train models on audio data, allowing for efficient processing of large datasets.
3) Time Series Forecasting: Mini-batch gradient descent can be applied to optimize models for time series forecasting, where data is often sequential and can be processed in smaller segments.
4) Generative Adversarial Networks (GANs): Mini-batch   gradient descent is frequently used in training GANs, where both the generator and discriminator networks are updated using small batches of data to improve training stability.


when to use which?

# use batch gradient descent when 
    - you have a small dataset that fits into memory and you want stable convergence.
    - your computational resources can handle the full dataset for each iteration.
    - you want to avoid the noise introduced by smaller batches.
    - you prioritize accuracy over speed in training.
    - your model is not too complex, and training time is not a major concern.

# use stochastic gradient descent when
    - you have a large dataset that does not fit into memory.
    - you want faster convergence and can tolerate some noise in the updates.
    - you are working in an online learning scenario where data arrives sequentially.
    - you want to quickly iterate and experiment with different model architectures.
    - your model is complex, and training time is a major concern.

    - you have a large dataset that can be processed in smaller batches.
    - you want to balance computational efficiency and convergence stability.
    - you have access to parallel computing resources (e.g., GPUs) that can process batches
    - you want to leverage the benefits of both batch and stochastic gradient descent. processing efficiently.
    - your model is complex, and you want to optimize training time without sacrificing too much accuracy.
    - you want to experiment with different batch sizes to find the optimal trade-off between speed and stability.
