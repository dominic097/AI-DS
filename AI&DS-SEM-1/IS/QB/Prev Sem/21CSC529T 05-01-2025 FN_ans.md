**Statistical Concepts Used:**

- Correlation Analysis
- Data Visualization

**Implementation:**

**Step 1: Understanding Customer Behavior**

**Formula: Mean Purchase Value**
```

```


**Insight:** High standard deviation ($31.78) indicates diverse purchasing patterns—need **segmentation**.

**Step 3: Correlation Analysis**

**Formula: Pearson Correlation**
```
r = Σ[(Xi - X̄)(Yi - Ȳ)] / √[Σ(Xi - X̄)² × Σ(Yi - Ȳ)²]
```

**Example:**

### **FUNCTIONALITY 3: A/B TESTING FOR FEATURE OPTIMIZATION**

**Scenario:** Test if new recommendation algorithm increases conversion rate

**Statistical Concepts:**
- Hypothesis Testing
- Two-Proportion Z-Test
- Confidence Intervals
- Statistical Power

**Setup:**
- Control Group (A): Old algorithm - 5,000 users
- Treatment Group (B): New algorithm - 5,000 users
- Metric: Conversion rate (purchases/visitors)

**Hypotheses:**
```
H₀: p₁ = p₂ (no difference in conversion rates)
H₁: p₁ ≠ p₂ (difference exists)
α = 0.05
```

**Results:**
```
Control: 250 conversions / 5,000 visitors = 5.0%
Treatment: 310 conversions / 5,000 visitors = 6.2%
```

**Statistical Test:**

**Formula: Two-Proportion Z-Test**
```
Z = (p̂₁ - p̂₂) / √[p̂(1-p̂)(1/n₁ + 1/n₂)]
```

**Calculation:**
```
Pooled proportion:
p̂ = (250 + 310) / (5000 + 5000) = 560/10000 = 0.056

SE = √[0.056(0.944)(1/5000 + 1/5000)]
SE = √[0.0529 × 0.0004]
SE = 0.0046

Z = (0.062 - 0.050) / 0.0046
Z = 0.012 / 0.0046
Z = 2.61
```

**Decision:**
```
Z = 2.61 > Z_critical = 1.96 → Reject H₀
P-value = 0.009 < 0.05
```

**Conclusion:** New algorithm **significantly increases** conversion by 1.2 percentage points (24% relative improvement).

**Business Impact:**
```
Additional conversions = 0.012 × 1,000,000 visitors/month = 12,000
Revenue increase = 12,000 × $65 avg = $780,000/month
```

- Time on site: 10 minutes
- Product views: 8
- Cart additions: 2

### **FUNCTIONALITY 5: RECOMMENDATION SCORING (COLLABORATIVE FILTERING)**

**Statistical Concept: Similarity Measures & Nearest Neighbors**

**Approach:** User-based collaborative filtering

**Formula: Cosine Similarity**
```
similarity(A,B) = (A · B) / (||A|| × ||B||)
```

**Example:**

**User Rating Matrix:**
```
         Prod1  Prod2  Prod3  Prod4  Prod5
User A:    5      4      0      3      5
User B:    4      5      0      4      4
User C:    0      0      5      0      3
```

**Calculate Similarity between User A and User B:**
```
A · B = (5×4) + (4×5) + (0×0) + (3×4) + (5×4)
      = 20 + 20 + 0 + 12 + 20 = 72

||A|| = √(25 + 16 + 0 + 9 + 25) = √75 = 8.66
||B|| = √(16 + 25 + 0 + 16 + 16) = √73 = 8.54

similarity(A,B) = 72 / (8.66 × 8.54) = 72 / 73.96 = 0.974
```

**High similarity (0.974)** → Users A and B have very similar tastes!

**Prediction Formula:**
```
Prediction(u,i) = r̄ᵤ + [Σ similarity(u,v) × (rᵥᵢ - r̄ᵥ)] / Σ|similarity(u,v)|
```

### **FUNCTIONALITY 6: TIME SERIES FORECASTING (DEMAND PREDICTION)**

**Statistical Concept: ARIMA (AutoRegressive Integrated Moving Average)**

**Purpose:** Forecast future product demand

**Formula: ARIMA(p,d,q)**
```
Yₜ = c + φ₁Yₜ₋₁ + φ₂Yₜ₋₂ + ... + θ₁εₜ₋₁ + θ₂εₜ₋₂ + ... + εₜ
```

Where:
- p = autoregressive order
- d = degree of differencing
- q = moving average order

**Example: Monthly Sales Forecast**

**Historical Data (Sales in thousands):**
```
Jan: 120, Feb: 130, Mar: 145, Apr: 140, May: 155, Jun: 165
```

**Step 1: Check Stationarity (Augmented Dickey-Fuller Test)**
```
H₀: Series has unit root (non-stationary)
ADF statistic = -3.8
Critical value = -2.9
Conclusion: Stationary (p < 0.05)
```

**Step 2: Fit ARIMA(1,1,1) Model**
```
Forecast for July = 170 ± 12 (95% CI: 158-182)
```

**Step 3: Evaluate Accuracy**

**Formula: Mean Absolute Percentage Error (MAPE)**
```
MAPE = (1/n) × Σ|Actual - Forecast| / Actual × 100%
```

```
MAPE = 3.2% (excellent accuracy)
```

### **FUNCTIONALITY 7: ANOMALY DETECTION (FRAUD DETECTION)**

**Statistical Concept: Outlier Detection**

**Method:** Z-Score and IQR Methods

**Formula: Z-Score**
```
Z = (X - μ) / σ
```

**Threshold:** |Z| > 3 indicates anomaly

**Example: Transaction Amount Monitoring**

**Normal transactions:** Mean = $75, SD = $25

**Suspicious transaction:** $250

**Calculation:**
```
Z = (250 - 75) / 25 = 175 / 25 = 7.0
```

**Decision:** |Z| = 7.0 > 3 → **Flag as potential fraud**

**Alternative: IQR Method**

**Formula:**
```
Q1 = 25th percentile
Q3 = 75th percentile
IQR = Q3 - Q1

Lower fence = Q1 - 1.5 × IQR
Upper fence = Q3 + 1.5 × IQR
```

**Example:**
```
Q1 = $50, Q3 = $100
IQR = $50
Lower fence = 50 - 75 = -$25 (use $0)
Upper fence = 100 + 75 = $175

Transaction of $250 > $175 → Anomaly
```

### Part (ii) - Handling Missing Values (5 Marks)

**Question:** How do you handle missing values while calculating mean of values?

---

#### **PROBLEM STATEMENT**

Missing values (NA, NULL, NaN) are common in real-world datasets. They affect statistical calculations, particularly the mean, because:
- Traditional mean formula requires **all values**
- Missing data can represent different scenarios
- Incorrect handling leads to **biased estimates**

---

#### **METHODS TO HANDLE MISSING VALUES**

---

### **METHOD 1: LISTWISE DELETION (COMPLETE CASE ANALYSIS)**

**Approach:** Remove all rows with any missing values

**Formula (Modified):**
```
μ = Σx / n_complete

Where n_complete = number of complete cases
```

**Example:**

**Dataset:**
```
Values: 10, 15, NA, 20, 25, NA, 30
```

**Calculation:**
```
Complete cases: 10, 15, 20, 25, 30

μ = (10 + 15 + 20 + 25 + 30) / 5
μ = 100 / 5
μ = 20
```

**Advantages:**
- Simple to implement
- Maintains data integrity
- No bias if **missing completely at random (MCAR)**

**Disadvantages:**
- **Loss of information** (2 values discarded)
- **Reduced sample size** (n = 5 instead of 7)
- Biased if **missing not at random (MNAR)**
- Low statistical power

**When to use:**
- Small proportion of missing data (<5%)
- Missing data is MCAR
- Large dataset (can afford loss)

---

### **METHOD 2: PAIRWISE DELETION**

**Approach:** Use all available data for each calculation

**Formula:**
```
μ = Σx_available / n_available
```

**Example:**

**For mean calculation:**
```
Values: 10, 15, NA, 20, 25, NA, 30

Available for mean: 10, 15, 20, 25, 30 (n=5)
μ = 100 / 5 = 20
```

**For variance calculation:**
```
May use different n if some values missing in (x-μ)² calculations
```

**Advantages:**
- Uses maximum available information
- Higher statistical power than listwise

**Disadvantages:**
- Different n for different calculations
- Can lead to inconsistencies
- Correlation matrices may not be positive definite

---

### **METHOD 3: MEAN IMPUTATION**

**Approach:** Replace missing values with the mean of available values

**Two-Step Process:**

**Step 1: Calculate mean from available data**
```
μ_initial = Σx_available / n_available
```

**Step 2: Replace missing values with μ_initial**
```
x_missing = μ_initial
```

**Step 3: Recalculate mean (optional)**
```
μ_final = Σx_all / n_total
```

**Example:**

**Original data:**
```
Values: 10, 15, NA, 20, 25, NA, 30
```

**Step 1:**
```
μ_initial = (10 + 15 + 20 + 25 + 30) / 5 = 20
```

**Step 2: Impute**
```
Imputed: 10, 15, 20, 20, 25, 20, 30
```

**Step 3:**
```
μ_final = (10 + 15 + 20 + 20 + 25 + 20 + 30) / 7
μ_final = 140 / 7
μ_final = 20 (same!)
```

**Mathematical Property:**
```
Mean imputation always yields the same mean as initial calculation
```

**Advantages:**
- Preserves sample size
- Simple to implement
- Mean unchanged

**Disadvantages:**
- **Underestimates variance** (reduces variability)
- Creates artificial data points
- Ignores relationships with other variables
- Weakens correlations

**When to use:**
- Quick approximation needed
- Missing completely at random
- Descriptive statistics only (not for inference)

---

### **METHOD 4: MEDIAN IMPUTATION**

**Approach:** Replace missing with median (robust to outliers)

**Formula:**
```
If n odd: Median = Middle value
If n even: Median = Average of two middle values
```

**Example:**

**Original:**
```
Values: 10, 15, NA, 20, 25, NA, 100 (note outlier!)
```

**Step 1: Calculate median**
```
Sorted available: 10, 15, 20, 25, 100
Median = 20 (middle value)
```

**Step 2: Impute**
```
Imputed: 10, 15, 20, 20, 25, 20, 100
```

**Step 3: Calculate mean**
```
μ = (10 + 15 + 20 + 20 + 25 + 20 + 100) / 7
μ = 210 / 7
μ = 30
```

**Comparison with mean imputation:**
```
If used mean imputation:
μ_initial = (10 + 15 + 20 + 25 + 100) / 5 = 34
Imputed: 10, 15, 34, 20, 25, 34, 100
μ_final = 238 / 7 = 34

Median imputation (30) less affected by outlier than mean (34)
```

**Advantages:**
- Robust to outliers
- Better than mean for skewed distributions

**Disadvantages:**
- Still reduces variability
- May not preserve relationships

---

### **METHOD 5: MODE IMPUTATION (FOR CATEGORICAL)**

**For categorical data or discrete values**

**Formula:**
```
Mode = Most frequent value
```

**Example:**
```
Values: Red, Blue, NA, Red, Green, NA, Red

Mode = Red (most frequent)
Imputed: Red, Blue, Red, Red, Green, Red, Red

Mean makes no sense for categorical data!
```

---

### **METHOD 6: FORWARD FILL / BACKWARD FILL (TIME SERIES)**

**For sequential data**

**Forward Fill:**
```
Original: 10, 15, NA, NA, 20, NA, 25
Forward:  10, 15, 15, 15, 20, 20, 25
```

**Backward Fill:**
```
Original: 10, 15, NA, NA, 20, NA, 25
Backward: 10, 15, 20, 20, 20, 25, 25
```

**Mean calculation:**
```
Forward Fill: μ = 120/7 = 17.14
Backward Fill: μ = 130/7 = 18.57
```

---

### **METHOD 7: INTERPOLATION**

**Linear interpolation between known values**

**Formula:**
```
x_missing = x₁ + (x₂ - x₁) × (position - pos₁) / (pos₂ - pos₁)
```

**Example:**
```
Values: 10, 15, NA, NA, 20, NA, 25

Position 3: x = 15 + (20-15) × (3-2)/(5-2) = 15 + 5×(1/3) = 16.67
Position 4: x = 15 + (20-15) × (4-2)/(5-2) = 15 + 5×(2/3) = 18.33
Position 6: x = 20 + (25-20) × (6-5)/(7-5) = 20 + 5×(1/2) = 22.50

Imputed: 10, 15, 16.67, 18.33, 20, 22.50, 25

μ = 127.50 / 7 = 18.21
```

**Advantages:**
- More realistic than constant imputation
- Preserves trends

**Disadvantages:**
- Still artificial
- Assumes linear relationship

---

### **METHOD 8: MODEL-BASED IMPUTATION**

**Use other variables to predict missing values**

**Approaches:**
- **Regression imputation**
- **K-Nearest Neighbors (KNN)**
- **Multiple Imputation by Chained Equations (MICE)**

**Example: Regression Imputation**

**Dataset:**
```
Age    Income    Spending (Y)
25     50000     20000
30     60000     NA
35     70000     28000
40     80000     32000
```

**Step 1: Build regression model using complete cases**
```
Model: Spending = β₀ + β₁(Age) + β₂(Income)

Using complete cases:
Spending = -5000 + 500(Age) + 0.3(Income)
```

**Step 2: Predict missing value**
```
For Age=30, Income=60000:
Spending = -5000 + 500(30) + 0.3(60000)
         = -5000 + 15000 + 18000
         = 28000
```

**Step 3: Calculate mean**
```
μ = (20000 + 28000 + 28000 + 32000) / 4
μ = 108000 / 4
μ = 27000
```

**Advantages:**
- Uses relationships between variables
- More accurate predictions
- Preserves variance better

**Disadvantages:**
- Complex to implement
- Requires complete cases for other variables
- Model may not be perfect

---

### **METHOD 9: MULTIPLE IMPUTATION**

**Most sophisticated approach**

**Process:**
1. Create **m** (typically 5-10) complete datasets with different imputations
2. Calculate mean for each
3. Pool results accounting for uncertainty

**Formula:**
```
μ_pooled = (1/m) × Σμᵢ

Where μᵢ = mean from imputation i
```

**Example (m=3):**

**Imputation 1:** Values become 10, 15, 18, 19, 25, 21, 30 → μ₁ = 19.71
**Imputation 2:** Values become 10, 15, 20, 22, 25, 23, 30 → μ₂ = 20.71
**Imputation 3:** Values become 10, 15, 17, 21, 25, 19, 30 → μ₃ = 19.57

**Pooled:**
```
μ_pooled = (19.71 + 20.71 + 19.57) / 3 = 20.00
```

**Advantages:**
- Accounts for imputation uncertainty
- Gold standard for missing data
- Produces valid statistical inference

**Disadvantages:**
- Computationally intensive
- Complex to implement
- Requires specialized software

---

### **METHOD 10: DOMAIN-SPECIFIC IMPUTATION**

**Use domain knowledge to fill missing values**

**Example: E-commerce**
```
Missing purchase amount:
- If cart abandoned: 0
- If return: negative of previous amount
- If partial delivery: proportion of original
```

| **Mean Imputation** | Low | Too low | Low | Quick approximation |

### **PRACTICAL EXAMPLE: COMPLETE WORKFLOW**

**Dataset: Student Scores**
```
Math: 85, 90, NA, 75, NA, 95, 80
Science: 80, 85, 70, NA, 90, 92, 78
```

**Step 1: Assess missing pattern**
```
Math: 2 missing (28.6%)
Science: 1 missing (14.3%)
Total complete: 4 cases (57%)
```

**Step 2: Choose method**
- Too much data loss for listwise (43% lost)
- Strong correlation between Math and Science expected
- **Use regression imputation**

```
Model (from complete cases):
```

- Imputation method used

- Check if conclusions change

**3. Consider mechanism:**

**4. Report limitations:**
- Acknowledge imputation in results
- Provide confidence intervals

Handling missing values when calculating means requires:
1. **Understanding** the missing data mechanism
2. **Choosing** appropriate imputation method
3. **Balancing** bias vs variance

## Question 3 (20 Marks)

### Part (i) - Bayes Theorem for Classification (12 Marks)

**Question:** State Bayes theorem. Show a use case which uses Bayes theorem for classification with its advantages and disadvantages.

---

#### **BAYES' THEOREM STATEMENT**

**Formula:**
```
P(A|B) = [P(B|A) × P(A)] / P(B)
```

**In words:**
```
Posterior = (Likelihood × Prior) / Evidence
```

**Where:**
- **P(A|B)** = Posterior probability (probability of A given B observed)
- **P(B|A)** = Likelihood (probability of observing B given A is true)
- **P(A)** = Prior probability (probability of A before observing B)
- **P(B)** = Evidence (probability of observing B)

---

**Expanded form (with mutually exclusive events):**
```
P(A|B) = [P(B|A) × P(A)] / [P(B|A)P(A) + P(B|A')P(A')]
```

---

#### **INTUITION**

Bayes' Theorem **updates beliefs** based on new evidence:

**Example:**
```
Prior: "30% chance of rain today" (based on season)
Evidence: "Dark clouds observed"
Likelihood: "80% of rainy days have dark clouds"
Posterior: "Higher chance of rain given dark clouds"
```

---

#### **USE CASE: EMAIL SPAM CLASSIFICATION**

**Problem:** Classify emails as "Spam" or "Not Spam" based on words in email

---

### **STEP 1: TRAINING PHASE (LEARNING FROM DATA)**

**Training Dataset:**

**100 Spam Emails:**
- 60 contain word "free"
- 45 contain word "winner"
- 80 contain word "click"

**200 Legitimate Emails:**
- 10 contain word "free"
- 2 contain word "winner"
- 15 contain word "click"

---

### **STEP 2: CALCULATE PRIOR PROBABILITIES**

**Formula:**
```
P(Spam) = Number of spam emails / Total emails
P(Not Spam) = Number of legitimate emails / Total emails
```

**Calculation:**
```
P(Spam) = 100 / 300 = 0.333 or 33.3%
P(Not Spam) = 200 / 300 = 0.667 or 66.7%
```

**Interpretation:** Before seeing email content, we assume **1 in 3 emails is spam**.

---

### **STEP 3: CALCULATE LIKELIHOODS**

**P(word | Spam)** = Probability word appears in spam email

**For word "free":**
```
P("free" | Spam) = 60/100 = 0.60
P("free" | Not Spam) = 10/200 = 0.05
```

**For word "winner":**
```
P("winner" | Spam) = 45/100 = 0.45
P("winner" | Not Spam) = 2/200 = 0.01
```

**For word "click":**
```
P("click" | Spam) = 80/100 = 0.80
P("click" | Not Spam) = 15/200 = 0.075
```

---

### **STEP 4: CLASSIFICATION (PREDICTION PHASE)**

**New Email Arrives:** "Congratulations! Click here for your free prize, winner!"

**Words:** "click", "free", "winner"

**Goal:** Calculate P(Spam | words) and P(Not Spam | words)

---

**NAIVE BAYES ASSUMPTION:**

**Assumes words are independent** (naive assumption):
```
P(words | Spam) = P("click"|Spam) × P("free"|Spam) × P("winner"|Spam)
```

---

**Calculate P(words | Spam):**
```
P(words | Spam) = 0.80 × 0.60 × 0.45
                = 0.216
```

**Calculate P(words | Not Spam):**
```
P(words | Not Spam) = 0.075 × 0.05 × 0.01
                    = 0.0000375
```

---

**Apply Bayes' Theorem:**

**For Spam:**
```
P(Spam | words) = [P(words | Spam) × P(Spam)] / P(words)

Numerator = 0.216 × 0.333 = 0.0719
```

**For Not Spam:**
```
P(Not Spam | words) = [P(words | Not Spam) × P(Not Spam)] / P(words)

Numerator = 0.0000375 × 0.667 = 0.000025
```

**Normalization (P(words)):**
```
P(words) = 0.0719 + 0.000025 = 0.072

P(Spam | words) = 0.0719 / 0.072 = 0.9986 or 99.86%
P(Not Spam | words) = 0.000025 / 0.072 = 0.0003 or 0.03%
```

---

**CLASSIFICATION DECISION:**

**Rule:** Classify as class with higher posterior probability

```
P(Spam | words) = 99.86% > P(Not Spam | words) = 0.03%

→ Classify as SPAM ✓
```

---

#### **DETAILED EXAMPLE WITH CALCULATIONS**

**Email 1:** "Meeting tomorrow at 3pm"

**Words:** "meeting", "tomorrow"

**Likelihoods (from training):**
```
P("meeting"|Spam) = 0.02, P("meeting"|Not Spam) = 0.40
P("tomorrow"|Spam) = 0.05, P("tomorrow"|Not Spam) = 0.25
```

**Calculate:**
```
P(words|Spam) = 0.02 × 0.05 = 0.001
P(words|Not Spam) = 0.40 × 0.25 = 0.10

P(Spam|words) ∝ 0.001 × 0.333 = 0.000333
P(Not Spam|words) ∝ 0.10 × 0.667 = 0.0667

Normalization:
P(Spam|words) = 0.000333 / 0.067 = 0.005 or 0.5%
P(Not Spam|words) = 0.0667 / 0.067 = 0.995 or 99.5%

→ Classify as NOT SPAM ✓
```

---

#### **ADVANTAGES OF NAIVE BAYES CLASSIFICATION**

**1. SIMPLICITY**
- Easy to understand conceptually
- Simple mathematical foundation
- Few parameters to estimate
- Easy to implement (50 lines of code)

**2. SPEED**
- **Training:** Very fast (just count frequencies)
- **Prediction:** Extremely fast (simple multiplication)
- Real-time classification possible
- **Time Complexity:** O(n × d) where n=samples, d=features

**3. SCALABILITY**
- Handles large datasets efficiently
- Works with high-dimensional data
- Can process millions of emails/documents
- Low memory requirements

**4. WORKS WITH SMALL TRAINING DATA**
- Requires relatively few training examples
- Makes reasonable assumptions
- Good generalization even with limited data
- Effective when data is scarce

**5. HANDLES MISSING DATA**
- Can work with incomplete feature sets
- Ignores missing features in calculation
- Robust to missing values
- No imputation needed

**6. PROBABILISTIC OUTPUT**
- Provides probability estimates
- Not just yes/no classification
- Confidence scores (P=99.9% vs P=51%)
- Can set custom thresholds

**7. INTERPRETABILITY**
- Easy to explain to non-technical users
- Can show which words contribute most
- Transparent decision process
- Auditable results

**8. WORKS FOR MULTIPLE CLASSES**
- Not limited to binary classification
- Can classify into many categories
- Example: Spam, Promotions, Social, Primary (Gmail)
- Naturally extends to multi-class

**9. BASELINE MODEL**
- Good starting point for comparison
- Often beats complex models
- Establishes performance benchmark
- Quick to test feasibility

**10. FEATURE IMPORTANCE**
```
High P(word|Spam) / P(word|Not Spam) ratio → Strong spam indicator

Example:
"free": 0.60/0.05 = 12× more likely in spam
"winner": 0.45/0.01 = 45× more likely in spam
"meeting": 0.02/0.40 = 0.05× (20× less likely in spam)
```

---

#### **DISADVANTAGES OF NAIVE BAYES CLASSIFICATION**

**1. INDEPENDENCE ASSUMPTION (MAJOR LIMITATION)**

**Problem:** Assumes features are conditionally independent given the class

**Reality:** Words often related!

**Example:**
```
"free" and "prize" often appear together in spam
"meeting" and "tomorrow" often together in legitimate emails

Naive Bayes treats them as independent → Incorrect probability estimates
```

**Impact:**
```
Actual: P("free" and "prize"|Spam) = 0.50
Naive Bayes: P("free"|Spam) × P("prize"|Spam) = 0.60 × 0.55 = 0.33

Underestimates correlation → Can hurt accuracy
```

**2. ZERO-FREQUENCY PROBLEM**

**Problem:** If word never appears in training data for a class, P(word|class) = 0

**Example:**
```
New word "cryptocurrency" appears in test email
Never seen in training spam or legitimate emails

P("cryptocurrency"|Spam) = 0/100 = 0
P("cryptocurrency"|Not Spam) = 0/200 = 0

P(words|Spam) = 0.60 × 0.45 × 0 = 0 (entire probability becomes zero!)
```

**Solution: Laplace Smoothing**
```
P(word|class) = (count + α) / (total + α × V)

Where α = smoothing parameter (typically 1), V = vocabulary size

P("cryptocurrency"|Spam) = (0 + 1) / (100 + 1×10000) = 1/10100 ≈ 0.0001

Small but non-zero probability
```

**3. POOR PROBABILITY ESTIMATES**

**Problem:** Actual probabilities often wrong (but rankings may be correct)

**Example:**
```
Naive Bayes says: P(Spam|words) = 99.9%
Reality: P(Spam|words) = 85%

Classification still correct (both > 50%), but confidence wrong
```

**Why it matters:**
- Can't use probabilities for decision-making with costs
- May trigger unnecessary actions (e.g., permanent email deletion)

**Solution: Calibration**
- Platt scaling
- Isotonic regression
- Adjusts probabilities post-training

**4. SENSITIVE TO IRRELEVANT FEATURES**

**Problem:** Includes all features, even irrelevant ones

**Example:**
```
Email timestamp included as feature
P(timestamp="3:42am"|Spam) calculated

But timestamp is irrelevant to spam classification!
Adds noise to model
```

**Impact:**
```
Too many irrelevant features → Lower accuracy
```

**Solution:**
- Feature selection
- Remove low-information features
- Use chi-square test or mutual information

**5. CANNOT LEARN FEATURE INTERACTIONS**

**Problem:** Multiplies individual probabilities, can't capture relationships

**Example:**
```
Phrase "not free" (negative sentiment)
Naive Bayes only sees:
- "not": P=0.30
- "free": P=0.60

Can't understand "not free" means opposite of "free"
```

**Solution:**
- Use n-grams (bigrams, trigrams)
- "not free" as single feature
- Captures some context

**6. DATA IMBALANCE SENSITIVITY**

**Problem:** Prior probabilities heavily influenced by class distribution

**Example:**
```
Training data: 95% Not Spam, 5% Spam

Prior: P(Spam) = 0.05 (very low)

Even with spam-like words, posterior may stay low
P(Spam|words) = (0.20 × 0.05) / 0.08 = 0.125 = 12.5%

Classified as Not Spam despite spam words!
```

**Solution:**
- Balanced training data
- Cost-sensitive learning
- Adjust priors manually

**7. CONTINUOUS FEATURES REQUIRE ASSUMPTIONS**

**Problem:** Bayes assumes discrete features; continuous features need distribution assumption

**Example:**
```
Email length (continuous): 250 words

Assumption: Length follows Gaussian distribution
P(Length=250|Spam) = (1/√(2πσ²)) × exp(-(250-μ)²/(2σ²))

If assumption wrong → Poor performance
```

**Solution:**
- Discretize continuous features
- Use Gaussian Naive Bayes variant
- Transform to categorical

**8. REQUIRES LARGE VOCABULARY**

### Part (ii) - When to Use Binomial and Poisson Distributions (8 Marks)

**Question:** When is it appropriate to use Binomial and Poisson distribution?

---

#### **BINOMIAL DISTRIBUTION**

**Definition:** Probability distribution for number of successes in **fixed number** of **independent** trials with **constant** probability of success.

---

### **WHEN TO USE BINOMIAL**

**Required Conditions (ALL must be met):**

**1. FIXED NUMBER OF TRIALS (n)**
```
Must know exactly how many trials
Cannot be random or unknown
```

**Examples:**
- Flip coin **10 times**
- Survey **100 customers**
- Inspect **50 products**

**2. BINARY OUTCOMES (Success/Failure)**
```
Each trial has exactly 2 outcomes
Success or Failure
Yes or No
Pass or Fail
```

**Examples:**
- Coin: Heads (Success) or Tails (Failure)
- Product: Defective (Success) or Good (Failure)
- Customer: Buy (Success) or Not Buy (Failure)

**3. INDEPENDENT TRIALS**
```
Outcome of one trial doesn't affect others
No memory between trials
```

**Examples:**
- ✓ Coin flips (each independent)
- ✓ Sampling with replacement
- ✗ Cards without replacement (dependent!)
- ✗ Contagious disease spread (dependent!)

**4. CONSTANT PROBABILITY (p)**
```
Probability of success same for all trials
Doesn't change over time or trials
```

**Examples:**
- ✓ Fair coin: p = 0.5 for all flips
- ✓ Defect rate: p = 0.05 constant
- ✗ Learning task: probability improves (not constant!)

---

### **BINOMIAL FORMULA**

**Probability Mass Function:**
```
P(X = k) = C(n,k) × p^k × (1-p)^(n-k)

Where:
n = number of trials
k = number of successes
p = probability of success
C(n,k) = n! / (k! × (n-k)!)
```

**Mean:**
```
μ = n × p
```

**Variance:**
```
σ² = n × p × (1-p)
```

**Standard Deviation:**
```
σ = √[n × p × (1-p)]
```

---

### **BINOMIAL EXAMPLES**

**Example 1: Quality Control**

**Scenario:** Factory produces light bulbs. Historical defect rate is 3%. Inspect 20 bulbs. What's probability exactly 2 are defective?

**Check conditions:**
- ✓ Fixed trials: n = 20
- ✓ Binary: Defective or Not
- ✓ Independent: Each bulb manufactured independently
- ✓ Constant p = 0.03

**Use Binomial!**

**Calculation:**
```
P(X = 2) = C(20,2) × 0.03² × 0.97¹⁸

C(20,2) = 20!/(2!×18!) = 190

P(X = 2) = 190 × 0.0009 × 0.554
P(X = 2) = 0.0947 or 9.47%
```

---

**Example 2: Marketing Campaign**

**Scenario:** Email campaign sent to 100 customers. Response rate is 5%. What's probability at least 8 respond?

**Solution:**
```
n = 100, p = 0.05

P(X ≥ 8) = 1 - P(X ≤ 7)
         = 1 - Σ P(X = k) for k = 0 to 7

Using binomial calculator or cumulative distribution:
P(X ≥ 8) = 1 - 0.867 = 0.133 or 13.3%
```

---

**Example 3: Exam Questions**

**Scenario:** Multiple choice exam, 10 questions, 4 options each. Student guesses all answers. What's probability they pass (≥6 correct)?

**Solution:**
```
n = 10, p = 0.25 (1/4 chance of guessing correctly)

P(X ≥ 6) = P(6) + P(7) + P(8) + P(9) + P(10)

Calculations:
P(X = 6) = C(10,6) × 0.25⁶ × 0.75⁴ = 0.0162
P(X = 7) = C(10,7) × 0.25⁷ × 0.75³ = 0.0031
P(X = 8) = C(10,8) × 0.25⁸ × 0.75² = 0.0004
P(X = 9) = C(10,9) × 0.25⁹ × 0.75¹ = 0.00003
P(X = 10) = C(10,10) × 0.25¹⁰ × 0.75⁰ = 0.000001

P(X ≥ 6) = 0.0197 or 1.97%

Very unlikely to pass by guessing!
```

---

#### **POISSON DISTRIBUTION**

**Definition:** Probability distribution for number of events occurring in **fixed interval** (time or space) when events occur **independently** at **constant average rate**.

---

### **WHEN TO USE POISSON**

**Required Conditions:**

**1. EVENTS IN FIXED INTERVAL**
```
Fixed time period: 1 hour, 1 day, 1 year
Fixed region: 1 km², 1 page, 1 volume
```

**Examples:**
- Customers arriving per hour
- Emails received per day
- Typos per page
- Earthquakes per year

**2. EVENTS ARE DISCRETE**
```
Can count events: 0, 1, 2, 3, ...
Cannot be fractional
```

**Examples:**
- ✓ Number of calls
- ✓ Number of accidents
- ✗ Time between events (use Exponential instead)

**3. EVENTS ARE INDEPENDENT**
```
One event doesn't affect probability of another
No clustering or patterns
```

**Examples:**
- ✓ Radioactive decay events
- ✓ Random customer arrivals
- ✗ Earthquake aftershocks (clustered!)
- ✗ Epidemic cases (contagious!)

**4. EVENTS OCCUR AT CONSTANT AVERAGE RATE (λ)**
```
Rate doesn't change over interval
Average rate known
```

**Examples:**
- ✓ Average 5 customers per hour (constant)
- ✗ Rush hour vs night (rate changes - need different λ)

**5. TWO EVENTS CANNOT OCCUR SIMULTANEOUSLY**
```
Infinitesimally small probability of 2+ events at exact same instant
```

**6. EVENTS ARE RARE**
```
Poisson is good for rare events
p small, n large in Binomial → Poisson approximation
```

---

### **POISSON FORMULA**

**Probability Mass Function:**
```
P(X = k) = (λ^k × e^(-λ)) / k!

Where:
λ = average rate (mean number of events)
k = actual number of events
e ≈ 2.71828
```

**Mean:**
```
μ = λ
```

**Variance:**
```
σ² = λ
```

**Standard Deviation:**
```
σ = √λ
```

**Unique Property:** Mean = Variance for Poisson!

---

### **POISSON EXAMPLES**

**Example 1: Call Center**

**Scenario:** Call center receives average 4 calls per minute. What's probability of receiving exactly 6 calls in a given minute?

**Check conditions:**
- ✓ Fixed interval: 1 minute
- ✓ Discrete events: Number of calls
- ✓ Independent: Callers don't influence each other
- ✓ Constant rate: λ = 4 per minute
- ✓ Rare: Can't receive multiple calls at exact same instant

**Use Poisson!**

**Calculation:**
```
λ = 4, k = 6

P(X = 6) = (4⁶ × e^(-4)) / 6!
         = (4096 × 0.0183) / 720
         = 74.96 / 720
         = 0.104 or 10.4%
```

---

**Example 2: Website Traffic**

**Scenario:** Website gets average 50 visitors per hour. What's probability of getting more than 60 visitors in next hour?

**Solution:**
```
λ = 50

P(X > 60) = 1 - P(X ≤ 60)

Using Poisson cumulative distribution:
P(X ≤ 60) ≈ 0.913

P(X > 60) = 1 - 0.913 = 0.087 or 8.7%
```

---

**Example 3: Typos in Book**

**Scenario:** Book averages 2 typos per 10 pages. What's probability of finding no typos in a 10-page chapter?

**Solution:**
```
λ = 2, k = 0

P(X = 0) = (2⁰ × e^(-2)) / 0!
         = (1 × 0.135) / 1
         = 0.135 or 13.5%
```

---

**Example 4: Rare Disease**

**Scenario:** Rare disease affects 1 in 10,000 people. In city of 100,000, how many cases expected?

**Solution:**
```
This is binomial with:
n = 100,000 (trials)
p = 0.0001 (probability)

But n large, p small → Use Poisson approximation!

λ = n × p = 100,000 × 0.0001 = 10

Expected cases: λ = 10

P(X = 10) = (10¹⁰ × e^(-10)) / 10!
          = 0.125 or 12.5%

P(X ≤ 5) = Σ P(k) for k=0 to 5 = 0.067 or 6.7%
```

---

#### **KEY DIFFERENCES**

| Aspect | Binomial | Poisson |
|--------|----------|---------|
| **Trials** | Fixed number (n) | Not fixed (interval-based) |
| **Parameters** | n (trials), p (probability) | λ (rate) |
| **Typical p** | Any value | Small (rare events) |
| **Variance** | n×p×(1-p) | λ (equals mean) |
| **Support** | 0 to n | 0 to ∞ |
| **Independence** | Between trials | Between events |
| **Examples** | Coin flips, Quality control | Call arrivals, Rare diseases |

---

#### **WHEN POISSON APPROXIMATES BINOMIAL**

**Rules of thumb:**
```
If n ≥ 20 and p ≤ 0.05 (or np ≤ 5):
Binomial ≈ Poisson with λ = np
```

**Example:**
```
Binomial: n = 100, p = 0.02
Calculate: P(X = 3) = ?

Exact (Binomial):
P(X = 3) = C(100,3) × 0.02³ × 0.98⁹⁷ = 0.182

Approximation (Poisson with λ = 100×0.02 = 2):
P(X = 3) = (2³ × e^(-2)) / 3! = 0.180

Very close! ✓
```

---

#### **PRACTICAL DECISION TREE**

```
Do you know the number of trials (n)?
├─ Yes → Fixed n?
│  ├─ Yes → Are outcomes binary?
│  │  ├─ Yes → Are trials independent?
│  │  │  ├─ Yes → Is p constant?
│  │  │  │  ├─ Yes → USE BINOMIAL
│  │  │  │  └─ No → Other distribution
│  │  │  └─ No → Other distribution
│  │  └─ No → Multinomial or other
│  └─ No → Not binomial
└─ No ↓

Are you counting events in an interval?
├─ Yes → Are events independent?
│  ├─ Yes → Is rate constant?
│  │  ├─ Yes → Are events rare?
│  │  │  ├─ Yes → USE POISSON
│  │  │  └─ No → May still work
│  │  └─ No → Non-homogeneous Poisson
│  └─ No → Other distribution
└─ No → Other distribution
```

---

#### **COMMON APPLICATIONS**

**BINOMIAL:**
1. Quality control (defect rates)
2. Clinical trials (treatment success)
3. A/B testing (conversion rates)
4. Customer surveys (response rates)
5. Pass/fail testing
6. Win/loss records
7. Market research (yes/no questions)
8. Genetics (trait inheritance)

**POISSON:**
1. Call center arrivals
2. Website traffic
3. Insurance claims
4. Manufacturing defects per unit time
5. Radioactive decay
6. Rare diseases
7. Network packet arrivals
8. Traffic accidents per day
9. Customer complaints per week
10. Typos per page

---

#### **VIOLATIONS AND ALTERNATIVES**

**If Binomial conditions violated:**
- **Non-constant p:** Beta-Binomial
- **Dependent trials:** Hypergeometric (sampling without replacement)
- **More than 2 outcomes:** Multinomial
- **Unknown n:** Negative Binomial

**If Poisson conditions violated:**
- **Rate changes:** Non-homogeneous Poisson
- **Events clustered:** Negative Binomial (overdispersed)
- **Events inhibit each other:** Other point processes

---

**SUMMARY:**

**Use BINOMIAL when:**
- Counting successes in fixed number of trials
- Key: n is known and fixed

**Use POISSON when:**
- Counting events in fixed interval
- Key: Rate (λ) is known, but number of events can vary

**Both require:**
- Independence
- Binary/discrete outcomes
- Constant probabilities/rates

---

#### **MEASURING SYMMETRY / SKEWNESS**

**Definition:** Skewness measures the **asymmetry** of a probability distribution.

---

### **TYPES OF SKEWNESS**

**1. SYMMETRIC (Zero Skewness)**
```
     _
    / \
   /   \
  /     \
 /       \
___________
  Mean = Median = Mode
```

**2. RIGHT-SKEWED (Positive Skewness)**
```
   _
  / \___
 /      ----____
/              -----___
_______________________
Mode < Median < Mean

Tail extends to right
```

**3. LEFT-SKEWED (Negative Skewness)**
```
        _
   ____/ \
---        \
   -----_  \
____________\_
Mean < Median < Mode

Tail extends to left
```

---

### **METHODS TO MEASURE SKEWNESS**

**1. KARL PEARSON'S COEFFICIENT (First)**
```
Skₚ = (Mean - Mode) / Standard Deviation
```

**2. KARL PEARSON'S COEFFICIENT (Second/Alternative)**
```
Skₚ = 3(Mean - Median) / Standard Deviation
```

**3. BOWLEY'S COEFFICIENT (Quartile-based)**
```
Skᵦ = (Q₃ + Q₁ - 2×Q₂) / (Q₃ - Q₁)
```

**4. MOMENT COEFFICIENT (Third Moment)**
```
Skₘ = μ₃ / σ³

Where μ₃ = third central moment
```

---

**Interpretation:**
```
Sk = 0: Perfectly symmetric
Sk > 0: Right-skewed (positive skew)
Sk < 0: Left-skewed (negative skew)

|Sk| < 0.5: Approximately symmetric
0.5 ≤ |Sk| < 1: Moderately skewed
|Sk| ≥ 1: Highly skewed
```

### Part (i) - Hypothesis Testing Process and Types (12 Marks)

**Question:** Explain the Hypothesis process and its types with a suitable example.

---

#### **HYPOTHESIS TESTING: COMPLETE FRAMEWORK**

**Definition:**
Hypothesis testing is a **statistical inference method** used to make decisions about population parameters based on sample data. It provides a systematic approach to determine whether observed effects are genuine or simply due to random chance.

---

#### **THE HYPOTHESIS TESTING PROCESS**

**STEP 1: STATE THE HYPOTHESES**

**Formula Framework:**
```
H₀ (Null Hypothesis): Statement of "no effect" or "status quo"
H₁ (Alternative Hypothesis): Statement of "effect exists" or "change occurred"
```

**Rules:**
- Hypotheses must be **mutually exclusive**
- Hypotheses must be **exhaustive** (cover all possibilities)
- H₀ typically contains equality (=, ≤, ≥)
- H₁ contains inequality (≠, <, >)

---

**STEP 2: CHOOSE SIGNIFICANCE LEVEL (α)**

**Formula:**
```
α = P(Type I Error) = P(Reject H₀ | H₀ is true)
```

**Common Values:**
- α = 0.05 (5%) - Standard in most research
- α = 0.01 (1%) - More stringent (medical research)
- α = 0.10 (10%) - Less stringent (exploratory studies)

**Interpretation:**
- α = 0.05 means "5% risk of incorrectly rejecting a true null hypothesis"

---

**STEP 3: SELECT TEST STATISTIC**

**Decision Tree:**
```
Population Parameter → Test Statistic
├─ Mean (μ)
│  ├─ σ known, n > 30 → Z-test
│  ├─ σ unknown, n < 30 → t-test
│  └─ σ unknown, n > 30 → t-test or Z-test
├─ Proportion (p) → Z-test for proportions
├─ Variance (σ²) → Chi-square test
└─ Two means → t-test or Z-test (paired/independent)
```

---

**STEP 4: COMPUTE TEST STATISTIC**

**Z-test Formula (for mean):**
```
Z = (x̄ - μ₀) / (σ/√n)

Where:
x̄ = Sample mean
μ₀ = Hypothesized population mean
σ = Population standard deviation
n = Sample size
```

**t-test Formula (for mean):**
```
t = (x̄ - μ₀) / (s/√n)

Where:
s = Sample standard deviation
df = n - 1 (degrees of freedom)
```

---

**STEP 5: DETERMINE CRITICAL REGION**

**For Two-Tailed Test (H₁: μ ≠ μ₀):**
```
Critical values: ±Z_(α/2) or ±t_(α/2, df)

Reject H₀ if: |Test Statistic| > Critical Value
```

**For Right-Tailed Test (H₁: μ > μ₀):**
```
Critical value: Z_α or t_(α, df)

Reject H₀ if: Test Statistic > Critical Value
```

**For Left-Tailed Test (H₁: μ < μ₀):**
```
Critical value: -Z_α or -t_(α, df)

Reject H₀ if: Test Statistic < Critical Value
```

---

**STEP 6: CALCULATE P-VALUE**

**Formula:**
```
P-value = P(observing test statistic as extreme or more | H₀ is true)

Decision Rule:
- If P-value ≤ α → Reject H₀ (Statistically significant)
- If P-value > α → Fail to reject H₀ (Not significant)
```

---

**STEP 7: MAKE DECISION**

**Decision Framework:**
```
IF Test Statistic falls in Critical Region OR P-value ≤ α:
   → Reject H₀
   → Accept H₁
   → "Evidence suggests effect exists"

ELSE:
   → Fail to reject H₀
   → "Insufficient evidence to support H₁"
   → NOTE: We NEVER "accept H₀" (only fail to reject it)
```

---

**STEP 8: STATE CONCLUSION**

**Template:**
```
"At α = [significance level], there is [sufficient/insufficient] evidence to
[support/reject] the claim that [contextual interpretation]."
```

---

#### **TYPES OF HYPOTHESIS TESTS**

---

**TYPE 1: BASED ON DIRECTIONALITY**

**1.1 Two-Tailed Test (Non-Directional)**

**Hypotheses:**
```
H₀: μ = μ₀
H₁: μ ≠ μ₀
```

**Critical Region:** Both tails (α/2 in each tail)

**Use When:** Testing for "difference" without direction
- "Is the mean different from 50?"
- "Is there a change in average sales?"

**Example:**
```
H₀: Average customer satisfaction = 7.0
H₁: Average customer satisfaction ≠ 7.0
```

---

**1.2 Right-Tailed Test (Upper-Tailed)**

**Hypotheses:**
```
H₀: μ ≤ μ₀
H₁: μ > μ₀
```

**Critical Region:** Right tail only (α in right tail)

**Use When:** Testing for "increase" or "greater than"
- "Has productivity increased?"
- "Is the new method faster?"

**Example:**
```
H₀: Average website load time ≤ 2.5 seconds
H₁: Average website load time > 2.5 seconds
```

---

**1.3 Left-Tailed Test (Lower-Tailed)**

**Hypotheses:**
```
H₀: μ ≥ μ₀
H₁: μ < μ₀
```

**Critical Region:** Left tail only (α in left tail)

**Use When:** Testing for "decrease" or "less than"
- "Has cost decreased?"
- "Is the new drug safer?"

**Example:**
```
H₀: Average defect rate ≥ 5%
H₁: Average defect rate < 5%
```

---

**TYPE 2: BASED ON NUMBER OF SAMPLES**

**2.1 One-Sample Test**

**Tests single sample against known/hypothesized population parameter**

**Z-test Formula:**
```
Z = (x̄ - μ₀) / (σ/√n)
```

**Example:** Testing if average height of students = 165 cm

---

**2.2 Two-Sample Test (Independent)**

**Tests difference between two independent groups**

**Formula (Equal Variances):**
```
t = (x̄₁ - x̄₂) / √[s²ₚ(1/n₁ + 1/n₂)]

Where pooled variance:
s²ₚ = [(n₁-1)s₁² + (n₂-1)s₂²] / (n₁ + n₂ - 2)
```

**Example:** Comparing test scores between Class A and Class B

---

**2.3 Paired Sample Test**

**Tests difference between paired observations**

**Formula:**
```
t = d̄ / (sᵈ/√n)

Where:
d̄ = Mean of differences
sᵈ = Standard deviation of differences
```

**Example:** Comparing blood pressure before vs. after medication

---

**TYPE 3: BASED ON PARAMETER TESTED**

**3.1 Test for Mean**
- Z-test (σ known)
- t-test (σ unknown)

**3.2 Test for Proportion**
```
Z = (p̂ - p₀) / √[p₀(1-p₀)/n]
```

**3.3 Test for Variance**
```
χ² = (n-1)s² / σ₀²
```

**3.4 Test for Comparing Variances**
```
F = s₁² / s₂²
```

---

#### **ERRORS IN HYPOTHESIS TESTING**

**TYPE I ERROR (α Error) - False Positive**

**Formula:**
```
α = P(Reject H₀ | H₀ is true)
```

**Example:**
- Null: Patient is healthy
- Reality: Patient is healthy
- Decision: Diagnose as sick
- **Consequence:** Unnecessary treatment

**Control:** Set lower α (e.g., 0.01 instead of 0.05)

---

**TYPE II ERROR (β Error) - False Negative**

**Formula:**
```
β = P(Fail to reject H₀ | H₀ is false)

Power of Test = 1 - β
```

**Example:**
- Null: Patient is healthy
- Reality: Patient is sick
- Decision: Diagnose as healthy
- **Consequence:** Missed diagnosis

**Control:**
- Increase sample size
- Increase α
- Use more powerful test

---

**ERROR TRADE-OFF TABLE:**

| Decision | H₀ True | H₀ False |
|----------|---------|----------|
| Reject H₀ | Type I Error (α) | Correct (Power = 1-β) |
| Don't Reject H₀ | Correct (1-α) | Type II Error (β) |

---

#### **COMPREHENSIVE EXAMPLE: NEW TEACHING METHOD**

**Business Context:**
A university claims that a new teaching method increases average exam scores from the traditional average of 70.

---

**STEP 1: STATE HYPOTHESES**

```
H₀: μ ≤ 70 (New method is not better)
H₁: μ > 70 (New method is better)

Type: Right-tailed test
```

---

**STEP 2: SET SIGNIFICANCE LEVEL**

```
α = 0.05 (5% significance level)
```

---

**STEP 3: COLLECT DATA**

**Sample Data:**
- Sample size: n = 36 students
- Sample mean: x̄ = 73.5
- Population SD: σ = 9 (from historical data)

---

**STEP 4: SELECT AND CALCULATE TEST STATISTIC**

**Since σ is known and n > 30, use Z-test:**

**Formula:**
```
Z = (x̄ - μ₀) / (σ/√n)
```

**Step-by-step Calculation:**

**1. Calculate Standard Error (SE):**
```
SE = σ/√n
SE = 9/√36
SE = 9/6
SE = 1.5
```

**2. Calculate Z-statistic:**
```
Z = (x̄ - μ₀) / SE
Z = (73.5 - 70) / 1.5
Z = 3.5 / 1.5
Z = 2.33
```

---

**STEP 5: DETERMINE CRITICAL VALUE**

**For right-tailed test with α = 0.05:**
```
Critical value: Z₀.₀₅ = 1.645

Critical Region: Z > 1.645
```

**Visual Representation:**
```
                      |     Rejection
                      |     Region
    _______________   |   /\
   /               \__|__/  \___
  /                  |
 /                   |
/                    |
─────────────────────┼────────────→
                   1.645      Z
                (Critical)

Our Z = 2.33 → Falls in rejection region
```

---

**STEP 6: CALCULATE P-VALUE**

**For Z = 2.33 (right-tailed):**
```
P-value = P(Z > 2.33)
P-value = 1 - P(Z ≤ 2.33)
P-value = 1 - 0.9901
P-value = 0.0099 or 0.99%
```

---

**STEP 7: MAKE DECISION**

**Using Critical Value Method:**
```
Test Statistic (2.33) > Critical Value (1.645)
→ Reject H₀
```

**Using P-value Method:**
```
P-value (0.0099) < α (0.05)
→ Reject H₀
```

---

**STEP 8: STATE CONCLUSION**

**Statistical Conclusion:**
"At the 5% significance level, there is **sufficient statistical evidence** to reject the null hypothesis."

**Contextual Conclusion:**
"The data provides **strong evidence** that the new teaching method increases average exam scores above 70. The improvement is statistically significant (Z = 2.33, p < 0.01)."

**Practical Interpretation:**
- The new method shows a mean improvement of 3.5 points
- This improvement is highly unlikely to occur by chance (only 0.99% probability)
- The university should consider implementing the new teaching method

---

**CONFIDENCE INTERVAL (Supplementary Analysis):**

**95% Confidence Interval for μ:**

**Formula:**
```
CI = x̄ ± Z_(α/2) × (σ/√n)
```

**Calculation:**
```
CI = 73.5 ± 1.96 × 1.5
CI = 73.5 ± 2.94
CI = (70.56, 76.44)
```

**Interpretation:** We are 95% confident that the true mean score with the new method lies between 70.56 and 76.44. **Since the entire interval is above 70, this confirms H₁.**

---

**POWER ANALYSIS:**

If true mean is 75, what's the probability of detecting it?

**Calculate β:**
```
Z_β = (μ₀ - μ₁) / SE + Z_α
Z_β = (70 - 75) / 1.5 + 1.645
Z_β = -3.33 + 1.645
Z_β = -1.685

β = P(Z < -1.685) = 0.046

Power = 1 - β = 1 - 0.046 = 0.954 or 95.4%
```

**Interpretation:** If the true mean is 75, we have a 95.4% chance of detecting this improvement.
