## Question 3

### Solve the 8-Puzzle Problem Using A* Search Algorithm

**Given Initial State:**
```
2 8 3
1 6 4
7 - 5
```

**Goal State:**
```
1 2 3
8 - 4
7 6 5
```

**Answer:**

#### A* Search Algorithm Components:

**Evaluation Function:** f(n) = g(n) + h(n)
- g(n) = Cost from start to current node
- h(n) = Heuristic (estimated cost to goal)
- f(n) = Total estimated cost

**Heuristic Used:** Manhattan Distance
- Sum of distances each tile is from its goal position

```python
import heapq

class PuzzleState:
    def __init__(self, board, parent=None, move="", g=0):
        self.board = board
        self.parent = parent
        self.move = move
        self.g = g  # Cost from start
        self.h = self.calculate_heuristic()
        self.f = self.g + self.h

    def calculate_heuristic(self):
        """Manhattan distance heuristic"""
        goal = [[1, 2, 3], [8, 0, 4], [7, 6, 5]]
        distance = 0
        for i in range(3):
            for j in range(3):
                if self.board[i][j] != 0:
                    goal_pos = self.find_goal_position(self.board[i][j], goal)
                    distance += abs(i - goal_pos[0]) + abs(j - goal_pos[1])
        return distance

    def find_goal_position(self, value, goal):
        for i in range(3):
            for j in range(3):
                if goal[i][j] == value:
                    return (i, j)
        return None

    def __lt__(self, other):
        return self.f < other.f

def solve_8_puzzle():
    initial = [[2, 8, 3], [1, 6, 4], [7, 0, 5]]
    goal = [[1, 2, 3], [8, 0, 4], [7, 6, 5]]

    start_state = PuzzleState(initial)
    open_list = []
    heapq.heappush(open_list, start_state)
    closed_set = set()

    while open_list:
        current = heapq.heappop(open_list)

        if current.board == goal:
            # Reconstruct path
            path = []
            while current:
                path.append((current.board, current.move, current.f))
                current = current.parent
            return path[::-1]

        closed_set.add(str(current.board))

        # Generate successors
        zero_pos = find_blank(current.board)
        for move, new_board in get_successors(current.board, zero_pos):
            if str(new_board) not in closed_set:
                new_state = PuzzleState(new_board, current, move, current.g + 1)
                heapq.heappush(open_list, new_state)

    return None

def find_blank(board):
    for i in range(3):
        for j in range(3):
            if board[i][j] == 0:
                return (i, j)
    return None

def get_successors(board, zero_pos):
    moves = []
    i, j = zero_pos
    directions = [('UP', -1, 0), ('DOWN', 1, 0), ('LEFT', 0, -1), ('RIGHT', 0, 1)]

    for move_name, di, dj in directions:
        ni, nj = i + di, j + dj
        if 0 <= ni < 3 and 0 <= nj < 3:
            new_board = [row[:] for row in board]
            new_board[i][j], new_board[ni][nj] = new_board[ni][nj], new_board[i][j]
            moves.append((move_name, new_board))

    return moves
```

#### Solution Steps:
1. Start with initial state, calculate f(n) = g(n) + h(n)
2. Add to open list (priority queue)
3. Pop state with lowest f(n)
4. If goal, reconstruct path
5. Otherwise, generate successors and add to open list
6. Repeat until goal found

**Important Points to Remember:**
- A* is optimal if heuristic is admissible (never overestimates)
- Manhattan distance is admissible for 8-puzzle
- Priority queue ensures lowest f(n) explored first
- Closed set prevents revisiting states
- g(n) increases by 1 for each move
- Path reconstruction uses parent pointers

### Part (ii): Alpha-Beta Pruning

**Answer:**

Alpha-Beta Pruning is an optimization of the Minimax algorithm that reduces the number of nodes evaluated in the search tree.

#### Main Idea:
- **Alpha (α):** Best value MAX can guarantee
- **Beta (β):** Best value MIN can guarantee
- Prune branches that won't affect final decision

#### How it works:
1. During MAX's turn, update α
2. During MIN's turn, update β
3. If β ≤ α, prune remaining children

#### Example:
```
       MAX
      /   \
    MIN   MIN
   / | \
  3  5  2
```

Process:
1. MAX explores first MIN node
2. MIN evaluates 3, β = 3
3. MIN evaluates 5, β remains 3
4. MIN returns 3, α = 3
5. When exploring second MIN:
   - If any child ≤ 3, prune remaining

**Benefits:**
- Reduces nodes evaluated from O(b^d) to O(b^(d/2))
- Same result as Minimax
- Allows deeper search

**Important Points to Remember:**
- Alpha-Beta doesn't change final result
- Move ordering affects efficiency
- Best-first ordering gives maximum pruning
- Alpha increases at MAX nodes
- Beta decreases at MIN nodes
- Prune when β ≤ α

### Part (iii): Bayes Theorem in Uncertain Knowledge Reasoning

**Answer:**

#### Bayes Theorem:
```
P(A|B) = P(B|A) * P(A) / P(B)
```

Where:
- P(A|B) = Posterior probability
- P(B|A) = Likelihood
- P(A) = Prior probability
- P(B) = Evidence

#### Example Application:
**Medical Diagnosis:**
- Disease D, Test T
- P(D) = 0.01 (1% have disease)
- P(T|D) = 0.95 (test 95% accurate for disease)
- P(T|¬D) = 0.05 (5% false positive)

**Calculate P(D|T):**
```
P(T) = P(T|D)*P(D) + P(T|¬D)*P(¬D)
     = 0.95*0.01 + 0.05*0.99
     = 0.0095 + 0.0495 = 0.059

P(D|T) = P(T|D)*P(D) / P(T)
       = 0.95*0.01 / 0.059
       = 0.161 or 16.1%
```

**Importance:**
- Updates beliefs with new evidence
- Foundation of Bayesian networks
- Handles uncertainty systematically

**Important Points to Remember:**
- Bayes theorem updates probabilities with evidence
- Prior × Likelihood = Posterior
- Requires knowing conditional probabilities
- Used in spam filters, medical diagnosis, etc.

### Part (i): BFS and DFS Algorithms

**Answer:**

#### Breadth-First Search (BFS):

**Algorithm:**
```python
from collections import deque

def BFS(graph, start, goal):
    queue = deque([(start, [start])])
    visited = {start}

    while queue:
        node, path = queue.popleft()

        if node == goal:
            return path

        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append((neighbor, path + [neighbor]))

    return None
```

**Properties:**
- Explores level by level
- Uses queue (FIFO)
- Complete: Always finds solution if exists
- Optimal: Finds shortest path
- Time: O(b^d), Space: O(b^d)

**Example:**
```
Graph:
    A
   / \
  B   C
 / \   \
D   E   F
```

BFS traversal from A: A → B → C → D → E → F

---

#### Depth-First Search (DFS):

**Algorithm:**
```python
def DFS(graph, start, goal, visited=None, path=None):
    if visited is None:
        visited = set()
    if path is None:
        path = []

    visited.add(start)
    path.append(start)

    if start == goal:
        return path

    for neighbor in graph[start]:
        if neighbor not in visited:
            result = DFS(graph, neighbor, goal, visited, path)
            if result:
                return result

    path.pop()
    return None
```

**Properties:**
- Explores depth first
- Uses stack (LIFO) or recursion
- Complete: Only in finite spaces
- Not optimal
- Time: O(b^d), Space: O(bd)

**Example:**
DFS traversal from A: A → B → D → E → C → F

**Comparison:**

| Feature | BFS | DFS |
|---------|-----|-----|
| Data Structure | Queue | Stack |
| Optimal | Yes | No |
| Complete | Yes | Only finite |
| Space | O(b^d) | O(bd) |
| Time | O(b^d) | O(b^d) |

**Important Points to Remember:**
- BFS uses queue, DFS uses stack
- BFS finds shortest path, DFS doesn't
- BFS requires more memory
- DFS better for deep trees
- Both have same time complexity

### Part (ii): A* Search for Delivery Robot

**Answer:**

**Scenario:** Robot navigating city grid with varying terrain costs.

**A* Implementation:**

```python
import heapq

def heuristic(node, goal):
    """Euclidean distance"""
    return ((node[0] - goal[0])**2 + (node[1] - goal[1])**2)**0.5

def a_star_search(grid, start, goal):
    """
    A* search with terrain costs
    grid[i][j] = cost of moving through cell (i,j)
    """
    open_set = []
    heapq.heappush(open_set, (0, start))

    came_from = {}
    g_score = {start: 0}
    f_score = {start: heuristic(start, goal)}

    while open_set:
        current = heapq.heappop(open_set)[1]

        if current == goal:
            return reconstruct_path(came_from, current)

        for neighbor in get_neighbors(current, grid):
            tentative_g = g_score[current] + grid[neighbor[0]][neighbor[1]]

            if neighbor not in g_score or tentative_g < g_score[neighbor]:
                came_from[neighbor] = current
                g_score[neighbor] = tentative_g
                f_score[neighbor] = tentative_g + heuristic(neighbor, goal)
                heapq.heappush(open_set, (f_score[neighbor], neighbor))

    return None

def get_neighbors(node, grid):
    neighbors = []
    directions = [(0,1), (1,0), (0,-1), (-1,0)]

    for dx, dy in directions:
        nx, ny = node[0] + dx, node[1] + dy
        if 0 <= nx < len(grid) and 0 <= ny < len(grid[0]):
            neighbors.append((nx, ny))

    return neighbors

def reconstruct_path(came_from, current):
    path = [current]
    while current in came_from:
        current = came_from[current]
        path.append(current)
    return path[::-1]
```

**Key Factors:**

1. **Heuristic Function:**
   - Euclidean distance: h = √((x₁-x₂)² + (y₁-y₂)²)
   - Manhattan distance: h = |x₁-x₂| + |y₁-y₂|
   - Must be admissible (never overestimate)

2. **Node Expansion:**
   - Expand node with lowest f(n) = g(n) + h(n)
   - g(n) = actual cost from start
   - h(n) = estimated cost to goal

3. **Path Cost Evaluation:**
   - Consider terrain costs (road, grass, water)
   - Update g_score when better path found
   - Use priority queue for efficiency

**Example Scenario:**
```
Start: (0, 0)
Goal: (4, 4)
Grid costs:
1 1 1 3 3
1 9 1 3 1
1 1 1 1 1
3 3 1 9 1
1 1 1 1 1
```

A* finds optimal path avoiding high-cost cells (9s).

**Important Points to Remember:**
- A* combines Dijkstra + greedy best-first
- Heuristic must be admissible for optimality
- Priority queue ensures efficiency
- Considers both path cost and estimated remaining cost
- Optimal if heuristic is admissible and consistent
